# Project Doc Initializer

## Description
通用项目 AI 文档初始化技能。通过分析项目特点，自动生成一套 AI 友好的文档体系，包括场景化导航、最佳实践规范和代码模板。适用于任何技术栈的项目。

## Triggers
- 初始化项目文档
- 创建 AI 开发文档
- 初始化 AI 文档
- 设置项目文档结构
- 项目文档初始化
- initialize project documentation
- create project docs
- AI documentation setup

## Usage
直接激活技能即可，无需额外参数。技能会自动分析当前项目并生成文档结构。

## Instructions

当用户激活此技能时，执行以下步骤：

### Step 1: 收集项目信息
通过读取以下文件了解项目：
- `package.json` - 技术栈和依赖
- 项目根目录结构
- 现有文档（README.md、docs/ 等）

### Step 2: 分析项目特点
识别：
- 技术栈（框架、语言、UI库等）
- 架构模式（SPA/MPA、微前端、多入口等）
- 项目结构特点
- 特有的开发规范
- 常用开发场景

### Step 3: 设计文档结构
基于分析结果设计：
1. **CLAUDE.md（主索引）**: 场景化导航 + AI工作原则 + 快速示例
2. **docs/ 目录**: 按功能拆分的详细文档

### Step 4: 生成文档内容
为每个文档编写核心内容：
- 场景化导航表
- 代码模板
- 规范说明
- 注意事项

### Step 5: 输出结果
1. 展示文件结构和预估行数
2. 提供完整的文档内容
3. 说明文档的使用方式

## Prompt Template

当创建具体项目的文档时，使用以下提示词模板：

```
你是项目文档规划专家。请为当前项目设计一套 AI 友好的文档体系。

## 项目信息
【从项目文件中自动分析获取】

## 任务要求

请按以下步骤设计：

### Step 1: 分析项目特点
识别技术栈、架构模式、项目结构、开发规范和常用场景。

### Step 2: 设计文档结构
设计原则：
1. CLAUDE.md（主索引）: 包含场景化导航
2. docs/ 目录: 按功能模块拆分的详细文档

### Step 3: 设计场景化导航表
根据项目实际设计导航表：

| 任务类型 | 文档路径 | 核心内容 |
|---------|---------|---------|
| 了解项目 | docs/PROJECT.md | 技术栈、目录结构 |
| 编写代码 | docs/CODE_STYLE.md | 编码规范 |
| ... | ... | ... |

### Step 4: 定义 AI 工作原则
1. 开发前必读
2. 按图索骥
3. 不懂再查

### Step 5: 编写文档内容
- CLAUDE.md: 场景导航 + AI工作原则 + 快速示例
- 各子文档: 规范 + 代码模板 + 注意事项

## 输出要求
1. 提供完整的文件列表
2. 编写每个文档的核心内容
3. 确保文档之间有交叉引用
4. 文档命名符合项目实际
5. 不限制技术栈，基于项目实际生成

## 格式要求
- CLAUDE.md 控制在 100 行以内
- 子文档单文件不超过 300 行
- 使用 Markdown 表格做导航
- 代码示例使用对应语言标注
```

## Notes

- 技能会自动检测项目类型（Vue/React/Node 等）
- 生成的文档结构可根据项目规模调整
- 建议先生成小规模试运行，再补充完善
- 文档路径使用相对路径，便于迁移
